# 最終課題：e-Statデータを用いたWebアプリ

## 1. アプリの概要
広告費と売上のデータを読み込み、その関係性や季節ごとの傾向を分析できるアプリです。

*   **URL**: [ここにStreamlit CloudのURLを貼る]
*   **GitHub**: [ここにGitHubのリポジトリURLを貼る]

## 2. データについて
*   **統計表名**: [※ここに使用したe-Statの統計表名などを記入してください]
*   **取得日**: 202X年X月X日
*   **各列の意味**:
    *   `prod_category`: 製品の種類
    *   `sales`: 売上（万円）
    *   `ad_expense`: 広告費（万円）
    *   `season`: 季節
*   **注意点**: 金額の単位は「万円」です。

## 3. 機能・構成
アプリは以下の3つの構成になっています。
1.  **データの絞り込み**: サイドバーで「製品カテゴリ」や「季節」を変更して、見たいデータを絞り込めます。
2.  **ダッシュボード**: データの件数や「総売上」「総広告費」などの重要な数字を計算して表示します。
3.  **グラフによる可視化**: 散布図と棒グラフを使って、データの傾向を視覚的に確認できます。

## 4. 授業で扱っていないUI部品（3つ以上）
このアプリでは、以下の新しい部品を調べて使ってみました。

*   **st.tabs（タブ）**
    *   **理由**: 情報を整理するために使いました。「分析画面」「元データ」「レポート」を分けることで、画面がごちゃごちゃしないようにしました。
    *   **場所**: メイン画面の上部
*   **st.metric（メトリック）**
    *   **理由**: 「総売上」などの大事な数字を、文字だけでなく大きく目立たせて表示したかったからです。
    *   **場所**: 分析タブの一番上
*   **st.expander（エクスパンダー）**
    *   **理由**: カラムの説明など、普段は隠しておきたい補足情報を入れるために使いました。
    *   **場所**: データタブの中

## 5. UI（操作性）の工夫
サイドバーを使うことで、条件を変えたときに結果がどう変わるかがすぐに分かるようにしました。
また、タブを使うことで、分析したいときとデータを確認したいときをワンクリックで切り替えられるようにしました。

## 6. ギャラリーの参照
*   **参考にしたアプリ**: Streamlit Galleryの "Sales Dashboard" 系アプリ
*   **参考にした点**: 画面の上部に重要な数字（KPI）を大きく並べるデザインが見やすかったので、自分のアプリでも `st.columns` と `st.metric` を使って真似してみました。
